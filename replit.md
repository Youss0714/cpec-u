# CPEC-U — Gestion Académique

## Overview
CPEC-U is a comprehensive academic management Progressive Web App (PWA) designed for educational institutions. It offers multi-role workflows for Admins (Directeur, Scolarité, Planificateur), Teachers, and Students, aiming to streamline academic operations, improve communication, and provide robust reporting and management tools. The project's vision is to enhance efficiency in educational administration and provide a modern, accessible platform for all stakeholders.

## System Architecture

### Monorepo Structure (pnpm workspaces)
The project utilizes a monorepo structure managed by pnpm workspaces:
- `artifacts/api-server/`: Express 5 REST API backend.
- `artifacts/cpec-u/`: React + Vite frontend PWA.
- `mockup-sandbox/`: UI component development sandbox.
- `lib/db/`: Drizzle ORM + PostgreSQL schema.
- `lib/api-spec/`: OpenAPI YAML specification.
- `lib/api-zod/`: Zod schemas generated from OpenAPI.
- `lib/api-client-react/`: React Query hooks generated by Orval.

### Tech Stack
- **Frontend**: React 19, Vite 7, Tailwind CSS 4, shadcn/ui, TanStack Query, Wouter, Framer Motion, Lucide React.
- **Backend**: Express 5, TypeScript, express-session, multer (file uploads), web-push.
- **Database**: PostgreSQL, Drizzle ORM, pg driver.
- **Validation**: Zod, Drizzle-Zod.
- **Tooling**: pnpm workspaces, tsx, esbuild, Orval codegen.

### Key Features and Technical Implementations
- **Multi-role System**: Supports Admin (Directeur, Scolarité, Planificateur), Teacher, and Student roles with distinct portals and permissions.
- **Academic Management**: Manages classes, subjects, teaching units, semesters, and annual promotions.
- **Grade Management**: Includes teacher grade entry, approval workflows, and retake session management. Each generated grade report (bulletin) embeds a unique QR code for verification.
- **Scheduling & Attendance**: Features timetable management with conflict detection, bulk schedule generation (`Programmation par période`), and attendance tracking with justification workflows. Role-based secure filtering for schedules is implemented for admins, teachers, and students.
- **Communication**: Integrated messaging with file attachments and Web Push notifications (VAPID).
- **PWA Capabilities**: Designed as a Progressive Web App with offline sync support and mobile-friendly design.
- **Jury Spécial**: Module for end-of-year special jury deliberations, traceability, bulletin updates, and PDF PV generation.
- **Digital Student ID Card (`Carte Étudiante Numérique`)**: Generates digital ID cards with QR codes, includes a public verification page, and supports admin generation/invalidation.
- **Teacher Evaluation (`Évaluation des Enseignants par les Étudiants`)**: Anonymous student evaluation module with a multi-step wizard, weighted scoring, and detailed admin/teacher reporting views. Ensures anonymity and requires a minimum evaluation threshold.
- **Parent Space (`Espace Parents`)**: Parent account system linked to student accounts, providing access to results, absences, and schedules. Automated notifications are sent to parents for absences and published semester results.
- **Payment Reminder Notifications (`Notifications de Rappel de Paiement`)**: Automated daily push notification system for fee reminders (J-7, J-3, J0, J+7) with a detailed log. Payment confirmations trigger immediate push notifications.
- **Individual Academic Tracking (`Suivi Académique Individualisé`)**: Provides admin and student views for academic performance tracking, including progression curves, subject-wise grades, absence rates, and automatic alerts for at-risk students based on multi-criteria analysis.
- **Multi-Year Comparative Dashboard (`Tableau de Bord Comparatif Multi-Années`)**: Analyzes academic performance over multiple academic years, including KPIs, success/failure rates, absence trends, and identification of high-failure subjects.
- **Grade Dispute Management (`Gestion des Réclamations`)**: Implements an official grade dispute workflow with configurable periods, student submission wizard, teacher response, and admin arbitration. Features immutable audit logs and push notifications at every stage.
- **Centralized PDF Service**: Client-side PDF generation engine (`jspdf`, `jspdf-autotable`) creating consistent documents (bulletins, attendance sheets, timetables, etc.) with institutional branding. An Admin "Centre de Documents" aggregates all available exports.

### Running the Application
The application runs as two separate processes:
- **API Server**: Runs on port 3001.
- **Frontend PWA**: Runs on port 8081, mapped to external port 80. Vite proxies `/api/*` requests to the API server.

### UI/UX Decisions
- Frontend utilizes React 19, Vite 7, Tailwind CSS 4, and shadcn/ui for a modern, responsive, and mobile-friendly design.
- Consistent institutional branding (header/footer, logo, gold accent bar) applied to all generated PDF documents.
- Interactive components like multi-step wizards for evaluations and grade disputes, and data visualization (radar charts, line charts, bar charts) for academic tracking and comparative reports.
- Color-coded scores and alerts are used for quick visual interpretation in evaluation and academic tracking modules.

## External Dependencies
- **PostgreSQL**: The primary database for all application data, managed by Drizzle ORM.
- **Web-Push (VAPID)**: For sending push notifications to users.
- **Multer**: For handling file uploads on the backend.
- **jspdf, jspdf-autotable**: Client-side libraries for PDF generation.
- **Orval**: Code generation tool used to create Zod schemas and React Query hooks from the OpenAPI spec.
- **Replit Built-in Database**: Utilized for PostgreSQL hosting.
